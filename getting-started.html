Getting Started with DIME

From installation to your first data flow in minutes

Step 1: Installation

DIME can be installed in multiple ways depending on your environment:

Option A: Windows Service

DIME.exe install
DIME.exe start

Option B: Docker Container

docker pull mriiot/dime:latest
docker run -d -p 9999:9999 -p 9998:9998 \
  -v /path/to/configs:/app/configs \
  mriiot/dime:latest

Option C: Direct Execution

dotnet run --project DIME/DIME.csproj -- --config /path/to/configs

Step 2: Basic Configuration

Create your first configuration file to connect to industrial devices. DIME uses YAML files for configuration:

Example: Connect to an OPC-UA Server

app:
  admin:
    enabled: true
    httpPort: 9999
    wsPort: 9998

sources:
  - name: myOpcServer
    connector: OpcUaSource
    scanInterval: 1000
    settings:
      endpointUrl: opc.tcp://localhost:4840
      userName: admin
      password: password
    items:
      - name: temperature
        path: ns=2;s=Machine.Temperature
      - name: pressure
        path: ns=2;s=Machine.Pressure

sinks:
  - name: dashboard
    connector: WebSocketSink
    includeFilter:
      - "**"

Step 3: Connect to Multiple Devices

DIME can connect to multiple devices and protocols simultaneously:

sources:
  # Modbus TCP Device
  - name: plc1
    connector: ModbusTcpSource
    scanInterval: 500
    settings:
      ipAddress: 192.168.1.100
      port: 502
    items:
      - name: coil_status
        address: 00001
        dataType: bool
      - name: register_value
        address: 40001
        dataType: uint16

  # MQTT Broker
  - name: mqttBroker
    connector: MqttSource
    settings:
      brokerAddress: mqtt://broker.hivemq.com
      topics:
        - sensors/+/temperature
        - sensors/+/humidity

  # SQL Database
  - name: productionDb
    connector: SqlServerSource
    scanInterval: 5000
    settings:
      connectionString: "Server=localhost;Database=Production;Trusted_Connection=true;"
      query: "SELECT * FROM CurrentProduction WHERE UpdateTime > @LastPoll"

Step 4: Data Transformation

Transform your data on-the-fly using built-in scripting engines:

Lua Script Example

sinks:
  - name: processedData
    connector: InfluxDbSink
    settings:
      url: http://localhost:8086
      database: production
    transform:
      engine: lua
      script: |
        -- Convert Celsius to Fahrenheit
        if msg.path:match("temperature") then
          msg.value = msg.value * 9/5 + 32
          msg.unit = "Â°F"
        end
        return msg

Step 5: Access the Web Portal

Once DIME is running, access the web management portal:

  1. Open your browser and navigate to http://localhost:9999
  2. You'll see the DIME dashboard with real-time connector status
  3. Click on "Data Explorer" to browse and visualize your data
  4. Use "Configuration" to modify settings through the web interface

The web portal provides:

  • Real-time connector monitoring
  • Interactive data visualization
  • Configuration management
  • System health metrics
  • Message flow timeline

Step 6: Advanced Features

Explore DIME's advanced capabilities:

Report by Exception (RBE)

items:
  - name: temperature
    path: ns=2;s=Machine.Temperature
    rbe:
      enabled: true
      deadband: 0.5  # Only report if change > 0.5
      heartbeat: 60000  # Send value every 60 seconds regardless

Multiple Configuration Files

Split your configuration across multiple files for better organization:

# configs/sources/plc.yaml
sources:
  - name: mainPLC
    connector: EthernetIpSource
    # ... configuration

# configs/sinks/database.yaml
sinks:
  - name: historian
    connector: PostgreSqlSink
    # ... configuration

# configs/main.yaml
app:
  admin:
    enabled: true
